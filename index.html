Git

Git is a version control software.

It allows users to revisit earlier versions of their files, compare changes between versions, undo changes and a lot more. A vesrion control system is a system that tracks and makes changes to files over time.

Its also allow users to revisit earlier versions of their files, compare changes between versions, undo changes and a lot more.

Different types of version control software include:

  1. Subversion
  2. Mercurial
  3. Git

Note that:In this discussion our main focus will be Git and GitHub.

Git Configuration

To config your git, we do this by providing your username and email address.

Configuring your name

  • Open command prompt on your laptop
  • Run the command: git config --global user.name "USERNAME"

    • Then replace "USERNAME" with your preffered name for example Lucy-Wanjiru

    git config --global user.name "Lucy-Wanjiru"

    Configuring your email

    • Open command prompt
    • Run the command: git config --global user.email "EMAIL_ADDRESS"

    Then replace "EMAIL_ADDRESS" with your email adress for example lucywanjiru@gmail.com

git config --global user.email lucywanjiru@gmail.com"

Verifying Configuration

We verify whether our Configurations are correct by:

  • Open the command prompt
  • To verify the username, run the command:

    git config user.name

  • The command should return the username that you set during the configuration process.
  • To verify the email address, run the command:

    git config user.email

Initializing a Git repository

What is a Git repository

A git repo is a workspace which tracks and manages files within a folder.

How to initialize a Git repository

To iniatialize a Git repository

  • Open your command line tool and navigate to your project directory.
  • Run the command:

    git init

  • The command will initialize a git repository in the project folder, this is where git will track and manage changes to files in your project.

NOTE:Initializing a git repository is something you do once per project. Running the command git init in a project that contains a git repository will make git initialize a repository within a repository.

Tip: use git status to confirm whether your project has a git repository or not, more about git status later. When you run the git status command and get the message "fatal: not a git repository..." being returned, then it means that the project does not have a git repository, you can go ahead and initialize one. If you get anything else back, then it means that the current directory contains a git repository, therefore no need to initialize one again.

Adding files to the staging area

The staging area also known as the index is an intermediate space where you can organize changes before committing them to the repository.

  1. To add changes to the staging area use git add command. And then commit them to the repository

  2. To add changes, run the command git add FILE_NAME to add changes to the staging area.
  3. Replace FILE_NAME with the name of the file, for example, in my project, I have a file story1.txt with changes that I would like to add to the staging area:

    4. git add story1.txt

  4. You can also add multiple files to the staging area, simply list all the files as shown:

    5. git add story1.txt story2.txt story3.txt

  5. If you want to add all the files in one go, you can use the period symbol or -A flag.

    6. git add -A or git add .

    Commiting changes

    Commiting will only work if there are changes in the staging area.

    1. To commit, we use the git commit command.
    2. Running git commit without passing any options will open a text editor (vi by default) for you to type in the message.

      3. NOTE: there is a -m flag that allows us to pass a commit message directly without having to open a text editor.

      git commit -m "MESSAGE"

    3. Replace MESSAGE with a short descriptive commit message of your choosing, for example:

      4. git commit -m "write story 1 to 3"

    Branches

    Branches are alternative timelines for a project.

    Why are branches important:

    They allow you us to create separate contexts where we can try new things or even work on multiple ideas in parallel.

    Creating Branches

    To create a branch, we use the command:

    git switch BRANCH_NAME

    Replace BRANCH_NAME with the name of the branch

    git switch story1

    if we want to make story1 the active branch.

    This will change the active branch from master to story1

    Viewing branches

    1. Use the command git branch to view your existing branches.
    2. The active branch will have a little asterisk symbol (*) beside it,and in some command line interfaces will be green in color.

    Switching branches

    • switch branches, we use the command:

      • git switch BRANCH_NAME

    • Replace BRANCH_NAME with the name of the branch, for example, if we want to make lucy-wanjiru the active branch (switching to it):

      • git switch lucy-wanjiru

    • This will change the active branch from master to lucy-wanjiru

    Below is an example

    Merging Branches

    To merge branches:

    First, we switch to the branch where we want to merge the current branch.

    We use the command:

    git merge BRANCH_NAME

    Deleting branches

    To delete a branch, we use the git branch command and pass the -d flag.

    git branch -d BRANCH_NAME

    Replace BRANCH_NAME with the name of the branch you want to delete, for example, to delete the branch lucy-wanjiru:

    git branch -d chapter1

    Below is an example:

    NOTE: Using -d flag to delete a branch that has not been merged might give us an error, if are sure we want to delete the branch, we can force git to delete it by passing -D flag to the git branch command:

    git branch -D BRANCH_NAME

    Then replace BRANCH_NAME

    git branch -D lucy-wanjiru
