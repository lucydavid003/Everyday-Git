Git Configuration

To config your git, we do this by providing your username and email address.

Configuring your name

  • Open command prompt on your laptop
  • Run the command: git config --global user.name "USERNAME"

    • Then replace "USERNAME" with your preffered name for example Lucy-Wanjiru

    git config --global user.name "Lucy-Wanjiru"

    Configuring your email

    • Open command prompt
    • Run the command: git config --global user.email "EMAIL_ADDRESS"

    Then replace "EMAIL_ADDRESS" with your email adress for example lucywanjiru@gmail.com

git config --global user.email lucywanjiru@gmail.com"

Verifying Configuration

We verify whether our Configurations are correct by:

  • Open the command prompt
  • To verify the username, run the command:

    git config user.name

  • The command should return the username that you set during the configuration process.
  • To verify the email address, run the command:

    git config user.email

Initializing a Git repository

What is a Git repository

A git repo is a workspace which tracks and manages files within a folder.

How to initialize a Git repository

To iniatialize a Git repository

  • Open your command line tool and navigate to your project directory.
  • Run the command:

    git init

  • The command will initialize a git repository in the project folder, this is where git will track and manage changes to files in your project.

NOTE:Initializing a git repository is something you do once per project. Running the command git init in a project that contains a git repository will make git initialize a repository within a repository.

Tip: use git status to confirm whether your project has a git repository or not, more about git status later. When you run the git status command and get the message "fatal: not a git repository..." being returned, then it means that the project does not have a git repository, you can go ahead and initialize one. If you get anything else back, then it means that the current directory contains a git repository, therefore no need to initialize one again.

Adding files to the staging area

The staging area also known as the index is an intermediate space where you can organize changes before committing them to the repository.

  1. To add changes to the staging area use git add command. And then commit them to the repository

  2. To add changes, run the command git add FILE_NAME to add changes to the staging area.
  3. Replace FILE_NAME with the name of the file, for example, in my project, I have a file story1.txt with changes that I would like to add to the staging area:

    4. git add story1.txt

  4. You can also add multiple files to the staging area, simply list all the files as shown:

    5. git add story1.txt story2.txt story3.txt

  5. If you want to add all the files in one go, you can use the period symbol or -A flag.

    6. git add -A or git add .
